# Hashi UI スタイルガイド

UI を「アプリらしく」揃えるための共通ルール(Issue #87)。**UI を追加・変更する
PR はこのガイドに準拠すること。** 定数・ヘルパーの実体は `hashi/style.py`。

## 1. 色

- **色コードの直書き禁止。** 必ず `hashi/style.py` の定数を使う
  (`setStyleSheet(f"color:{style.FG_MUTED};")` のように)。
- パレットは Fusion ダーク(main.py)と `style.py` で一致させる。片方だけ変えない。
- セマンティックカラーは意味が固定:
  - `WARN`(#d0a050): 取り返しがつきにくい操作の注意書き。`warning_label()` を使う
  - `ERROR`(#e0655f): エラー表示・危険な状態・閉じる
  - `OK`(#77c777): 成功・安全の表示
- パレット(参考デザイン TransTerm、Issue #111): 背景 `BG`=#1e1f24 /
  入力欄 `BG_BASE`=#191a1f / チップ・ボタン `BG_RAISED`=#2a2b33 /
  文字 `FG`=#e8e8ec / 補足 `FG_MUTED`=#9a9ba6 / アクセント `ACCENT`=#4f8cff
  (hover #7caaff) / 枠線 `BORDER`=#3a3b44。
- 小型ボタンの並ぶツールバーは `style.chip_style()`(角丸 6px・hover 背景・
  押下でアクセント枠、危険スイッチは `danger=True` で DANGER_BG)を使う。
  - 補足・注記は `FG_MUTED`。`muted_label()` / `muted_span()` を使う
- 新しい「意味」の色が必要になったら、まず style.py に定数と用途を定義してから使う。

## 2. 余白・寸法

- **8px グリッド**: マージン・スペーシングは 8 の倍数(8 / 16 / 24)。
- ダイアログの最小幅は 3 段階のみ: `DIALOG_S`(420)/ `DIALOG_M`(520)/
  `DIALOG_L`(640)。中間の独自値を増やさない。
- ウィンドウの初期サイズは `windowfit.fit_to_screen` を必ず通す(画面外対策 #63)。

## 3. ダイアログの型

すべての設定・操作ダイアログは次の構成に揃える:

```
[フォーム (QFormLayout)]
[注意書き (warning_label / muted_label)]   ← 必要な場合のみ
[ボタン行 (QDialogButtonBox)]
```

- ボタンは **肯定(「適用」「保存」など動詞)= AcceptRole、「キャンセル」= RejectRole** の
  2 つを基本とする。OK/Yes などの曖昧なラベルは使わない。
- **破壊的・不可逆な操作は `DoubleCheckDialog`(確認語入力)を必須**とする
  (削除の 2 段階確認、`change` 入力など既存の流儀を踏襲)。
- 現在の状態を変更するダイアログは、**冒頭に現在値を明示**する
  (例: sshd 設定の「現在の状態: パスワード認証は有効」 #73)。

## 4. フィードバック(通知の使い分け)

- **一時的な進行・完了**: ステータスバー `showMessage(…, 5000)`(5 秒)。
- **ユーザーが読むべき結果・エラー**: `QMessageBox`(information / warning)。
  本文には「何が起きたか」だけでなく「次に何をすべきか」を書く。
- **ターミナル内の文脈通知**: トースト(既存の `_toast`)。角丸は `TOAST_RADIUS`。
- 静かに失敗しない。自動処理が失敗・ロールバックしたら必ずユーザーに伝える(#61 の教訓)。

## 5. 文言(日本語 UI)

- メニュー・ボタンは**動詞で終える**(「〜を追加…」「〜を変更…」)。
  続きのダイアログが開く項目は末尾に「…」を付ける。
- 注意書きは「何が危険か」→「アプリが何で守るか」の順で書く
  (例:「誤ると切断されます(適用前にバックアップし、失敗したら自動で戻します)」)。
- 未検証・制限事項は隠さず明記する(プロジェクトの流儀)。
- カタカナ語の長音は付ける(「サーバー」「ユーザー」)。

## 6. アイコン・装飾

- 絵文字は**機能の目印としてボタン/見出しに 1 個まで**(例: 🔑 パスワード送信)。
  文中に散りばめない。
- 独自の `setStyleSheet` での装飾は最小限にし、色は必ず style.py 経由。
  パレット(Fusion)が効くところはパレットに任せる。

## 7. 移行ルール

- 既存コードの直書き色は、**そのファイルを触る PR のついでに style.py 定数へ置換**する
  (ボーイスカウト方式)。専用の巨大置換 PR は作らない(コンフリクト源になるため)。
