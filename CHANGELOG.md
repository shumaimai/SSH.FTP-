# 変更履歴

このプロジェクトは [Semantic Versioning](https://semver.org/lang/ja/) に緩く従います。

## [Unreleased]
### 追加
- **接続モードの選択: ターミナルのみ / ファイルのみ / 両方(Issue #112)**。
  ランチャーの右クリックメニューから、SSH ターミナルだけ・SFTP ファイルだけ・
  両方を選んで接続できる。ターミナルのみのときは SFTP チャネルを張らず、
  ファイルのみのときはシェルを開かない(軽量)。無効な側のトグル・
  パスワード送信・セッションログは自動で無効化。

### 変更
- **デザインの仕上げ(Issue #113 の続き)**。設定ダイアログに**テーマプレビュー**を追加
  (選択に追従してターミナルの見本を表示)。ランチャーに**空状態プレースホルダ**
  (接続先ゼロ / 検索一致なしで案内)。ファイルブラウザ上部のナビゲーションボタンを
  チップ型(QToolButton)にしてセッションのツールバーと統一。ホスト鍵 / 認証情報入力
  ダイアログの体裁を整理(余白・枠・見出し・既定ボタンのアクセント化)。
- **全体の意匠を刷新(Issue #113、参考デザイン TransTerm 第 2 弾)**。
  アプリ全体へ共通スタイルシート `style.app_stylesheet()` を適用し、タブ / ボタン /
  入力欄 / リスト / メニュー / スクロールバー / ステータスバーの質感を統一
  (丸み・フォーカスリング・スリムスクロールバー・アクセント選択色)。
  セッション画面に **チップ型ツールバー**(表示トグル + パスワード送信)と
  **情報ステータスバー**(接続先 / ネゴシエート済み暗号スイート / 文字コード /
  転送進捗)を追加。ランチャーの「新しい接続」を主ボタン(アクセント)に。
  暗号スイートは `SshSession.security_summary()` が接続後の transport から取得。
- **意匠の詰め(Issue #113 第 2 弾)**。ダイアログの決定ボタンを自動でアクセント色に統一。
  接続中は丸みカード + アクセントプログレスで表示、切断時は警告バナー(再接続は主ボタン)。
  トーストをフェードイン/アウトに。ターミナル既定テーマ **Hashi**(アプリのパレットに調和:
  背景 #1a1b20 / 前景・カーソルをアクセントに合わせる)を追加し既定に採用。
- **ブラウザ風タブ UI へ移行(Issue #115)**。これまでの「1 接続 = 1 ウィンドウ」を
  やめ、1 つのウィンドウ(AppWindow)に「サーバー一覧」タブと各接続タブを並べる
  ブラウザ風の構成にした。`+` で一覧を開き、ダブルクリック / 右クリックメニューで
  接続すると新しいタブ(SessionPage)が開く。タブの `✕` で個別に切断・クローズ。
  切断されたタブは見出しに `⚠` が付く。セッションメニューは現在のタブに追従する。
- **UI パレットを刷新(Issue #111、参考デザイン TransTerm)**。アプリ全体の配色を
  背景 #1e1f24 / アクセント #4f8cff / 文字 #e8e8ec 系へ更新。`hashi/style.py` と
  `main.py` の Fusion パレットを同一ソースから適用(テストで一致を強制)。
  チップ型ボタンのヘルパー `style.chip_style()` を追加。dialogs / mainwindow /
  editor に残っていた色の直書きを style 定数へ置換。

## [0.7.0] - 2026-07-18
### 追加
- **ファイルの「実行」メニュー(Issue #98、`hashi/fileactions.py`)**。SFTP ブラウザで
  ファイルを右クリック →「実行」から、種類に応じたコマンドを選べる:
  docker-compose(起動/停止/状態/ログ)、Dockerfile(ビルド)、.py / .sh / .jar の実行、
  .service の systemctl 操作、Makefile / package.json / requirements.txt、
  tar / zip の展開など。**コマンドはターミナルへ入力されるだけで、実行(Enter)は
  必ず人間が押す**(スニペットと同じ安全思想)。パスはシェルクォート済み、
  `{{tag}}` などの追加変数は入力ダイアログで指定。

### 修正
- **ターミナルを縮めて入力行が折返した後に広げると、入力位置と表示が崩れる
  バグ(Issue #100、#96 実機報告の根本原因)**。pyte はリフロー(行の再折返し)
  非対応のため、シェルの想定と画面の行構成がずれていた。自動折返しで生まれた
  継続行を追跡し、リサイズ時に**カーソルの論理行(入力行)を新しい幅で
  再折返し**するようにした(第 1 段。過去出力の全画面リフローは対象外)。
  実 sshd + 実 bash で縮小→拡大→入力継続→実行の通しを検証済み。
### 修正
- **ターミナル入力中に Backspace / Delete / F2 / F5 がファイルブラウザに横取り
  されるバグ(v0.6.0 実機で発生)**。ショートカットが既定のウィンドウ全体
  スコープだったため。ファイルツリーにフォーカスがあるときだけ効くよう限定し、
  さらにターミナルはフォーカス中の打鍵を ShortcutOverride で自分のものと宣言する。

### 変更
- **ターミナルのテーマ / フォント設定を開いているセッションへ即時反映**。
  従来は「新しい接続から反映」で再接続が必要だった(オーナーのフィードバック)。

## [0.6.0] - 2026-07-17
### 追加
- **スニペット(よく使うコマンド、Issue #83)**。登録したコマンドをメニュー /
  ターミナル右クリックからワンクリック送信。本文の `{{変数名}}` は送信時に入力。
  自動送信はしない設計(sudo ボタンと同思想)で、Enter 付き送信は明示フラグ。
- **キープアライブと自動再接続(Issue #84)**。キープアライブ間隔を設定可能に
  (既定 30 秒)。切断を検知すると「再接続」バーを表示し、設定で指数バックオフの
  自動リトライ。再接続時はターミナル / SFTP ワーカー / 権限管理を新しい接続へ
  張り替え、転送キューは維持する。
- **セッションログの自動保存(Issue #85、PuTTY logging 相当)**。受信した出力を
  プレーンテキストでローカルへ追記(入力は記録しない)。設定でオン/オフと保存先、
  メニューから開始/停止。書き込み失敗はログのみ停止しセッションは継続。
- **SSH エージェントフォワーディング(Issue #86、ssh -A 相当)**。接続設定で
  明示オプトイン(既定オフ・リスクの注意書き付き)。`~/.ssh/config` の
  `ForwardAgent yes` も反映。実 sshd で転送鍵の一覧表示を確認済み。
- **ターミナルの配色テーマとフォント選択(Issue #78)**。設定ダイアログで
  6 テーマ(One Half Dark / Solarized Dark / Solarized Light / Monokai /
  Dracula / Nord、`hashi/themes.py`)と等幅フォントを選択できる。
  新しい接続/ウィンドウから反映。
- **ランチャー強化(Issue #81)**。サーバーが増えても迷子にならない接続先一覧に:
  - 🔍 インクリメンタル検索(名前 / ホスト / ユーザー名 / タグで絞り込み)
  - プロファイルに**タグ**と**色マーカー**(プリセット 7 色)を追加。接続設定
    ダイアログで編集でき、一覧に色ドットとタグを表示
  - **最終接続日時**を記録し「3 日前」のような相対表示。最終接続が新しい順に並ぶ
  - 一覧は 2 行表示(名前 / user@host:port + タグ + 最終接続)に刷新
  - 旧 profiles.json はそのまま読める(後方互換をテストで固定)
### 追加(開発基盤)
- **UI 共通スタイル基盤(Issue #87)**。`docs/ui-style-guide.md`(色・余白・

### 変更
- **SFTP ブラウザのツールバーを再構成(Issue #96)**。アップロードは常時表示、
  上へ/更新はアイコン化、低頻度の操作(ホーム / 隠しファイル / 権限無視 /
  転送キュー表示)はプルダウンメニューへ集約。ダウンロード / 新規フォルダ /
  削除は右クリックメニューに一本化。最小幅が 544px → 240px になり、スプリッタで
  ファイル側を十分小さくできるようになった。転送キューは下部に常時文字表示。

### 修正
- **リサイズ後にカーソルが新しい列数の外に残り、入力位置がずれるバグ(Issue #72
  の続き)**。pyte の resize は幅縮小時にカーソル x を追従させないため、resize
  成功後に `ensure_hbounds/vbounds` で新グリッド内へ clamp する(Windows Server
  実機 + 実 bash で検証済み)。
- **ターミナルのリサイズ / 表示切替で入力位置(カーソル)が乱れることがあるバグ
  (Issue #72)**。pyte スクリーンの resize が失敗しても描画グリッドと PTY だけ
  新サイズに更新され、シェル(新幅で折返す)と pyte(旧幅)がずれていた →
  3 者(スクリーン / グリッド / PTY)のサイズが常に一致するよう、スクリーンを
  変えられなかったときは何も変更しないように修正。履歴スクロール中のリサイズは
  最下段へ戻してから適用する。**実機での再現解消の確認はオーナー待ち**。

### 変更
- **SSH サーバー設定変更ダイアログに現在の実効状態を明示(Issue #73)**。
  パスワード認証の有効/無効と待受ポートを、`sshd -T` の実効値(Include /
  sshd_config.d の優先順を解決済み)としてダイアログ冒頭に表示。Match ブロックで
  個別上書きしている構成では実挙動が異なりうる旨の注意書き付き。

### 修正
- **IP 固定を 2 回実行すると両方の IP が有効になるバグ(Issue #71、実機で発生)**。
  - バックアップに前回の Hashi ドロップインが混入し、ロールバック時に「元の設定」
    ではなく前回の静的 IP が復元されていた → バックアップから `90-hashi.yaml` を
    除外するよう修正。
  - 再実行時は確認ダイアログに「前回の固定設定を置き換える」ことを明示。
  - 確定後の掃除用の新 IP 接続が張れない場合、旧接続から遅延ジョブ
    (nohup + sleep)で残留アドレスの削除を仕掛けるフォールバックを追加。
    それも不可なら手動削除コマンドを結果ダイアログに明示。

### 変更
- **SSH ポート変更の自動化(Issue #62)**。ポート変更が疎通確認まで成功したら、
  接続プロファイルのポート番号を自動で更新する(次回から新ポートで接続)。
  複数ポートを待ち受けている場合に備え、ダイアログで「現在の待受ポート」から
  基準を選択できるようにした。`change` 入力の 2 段階確認・バックアップ・
  自動ロールバックは従来どおり。
- **サーバー IP 固定の改良(Issue #61、実機フィードバック反映)**。
  - `netplan apply` が剥がしてくれない**残留アドレスを明示的に掃除**: ロールバック時は
    新 IP を `ip addr del`、成功確定後は旧 IP を新 IP 側の接続から削除(「IP の二重登録」
    の解消)。ループバックには触れない。
  - **成功時に接続プロファイルの IP を自動更新**し、旧 IP のセッションウィンドウを閉じる
    (次回から新 IP で接続できる)。
  - **ロールバックが起きたことを通知**: 番兵発火の痕跡をサーバーに残し、次回 IP 固定を
    開いたときに「前回は確定されず元に戻った」ことを表示(従来は静かに戻り成功と誤解できた)。
    確定とタイマー発火が競合した場合も成功と誤認しない。
  - ダイアログ改善: ゲートウェイは現在のデフォルトルートから自動プリフィル、DNS は
    1.1.1.1 を既定、自動ロールバックの既定を 20 秒に(10〜600 秒で調整可)。
  - `change` 入力の 2 段階確認・netplan 限定は従来どおり。**実 Ubuntu での通し検証は未実施**。

## [0.4.0] - 2026-07-15
### 追加
- **外部アプリ自動アップロードのオン/オフ設定(Issue #54)**。「関連付けアプリで開く」で
  開いたファイルの変更を自動でリモートへ再アップロードする挙動(実装済み)を、設定ダイアログの
  「関連付けアプリで開いたファイルの変更を自動アップロード」で無効化できるようにした
  (既定はオン)。オフ時はステータスに「自動保存オフ」と明示。**Windows 実機での挙動
  (アプリのファイルロック・別名保存)は未検証**。
- **サーバーの静的 IP 設定(GUI、Issue #45)**。セッションウィンドウの「セッション」メニューに
  「サーバーの IP を固定…」を追加。インターフェース一覧・アドレス(CIDR)・ゲートウェイ・DNS・
  自動ロールバック秒数を入力し、2 段階確認のうえ適用する。疎通確認(新 IP への別接続)は
  ワーカースレッドで実行。netplan 以外は実行前に拒否。**実 Ubuntu での通し適用は未検証**
  (offscreen のダイアログ/ワーカー配線テストまで)。
- **サーバーの静的 IP 設定(コア、Issue #45、`hashi/netadmin.py`)**。netplan(Ubuntu Server)
  限定で、静的 IP を安全に適用するコアロジック。ネットワーク設定の失敗は SSH ごと切断され
  復旧が重いため、**自動ロールバック**を組み込む: バックアップ → ドロップイン YAML を配置 →
  `netplan generate` で構文検証 → 「一定時間内に確定しなければ元へ戻す」番兵ジョブを仕掛けて
  `netplan apply` → 新 IP へ別接続で疎通確認 → OK なら確定、失敗なら即ロールバック。netplan
  以外の環境は検出したら明示エラーで拒否(黙って触らない)。設定ファイルは SFTP 一時書き込み
  + `sudo install`(内容を sudo の stdin に通さない)。**実 Ubuntu での通し検証は未実施**
  (フェイクセッションのユニットテストまで。GUI 配線は次段)。
- **アカウント同期(クラウド同期、Issue #44、`hashi/cloudsync.py`)**。接続情報を
  Google Drive の appDataFolder(アプリ専用の隠し領域)に置いて別端末と同期できる。
  **専用サーバー不要でクライアント完結**。置くのは #42 のバンドルで、**端末側でマスター
  パスフレーズ由来の鍵(scrypt + Fernet)で丸ごと暗号化**してからアップロードするため、
  クラウド側には暗号化済み blob しか渡らない(E2E)。競合は「最後に書いた方が勝ち」で、
  取り込み前に手元をローカルへ自動バックアップする。保存先は `SyncBackend`(get/put)で
  差し替え可能(Google Drive 実装を同梱、任意の WebDAV/S3 も同じ形で載る)。Google の
  クライアントライブラリは任意依存(`requirements-cloud.txt`)で、未導入なら同期メニュー
  使用時に案内するだけで他機能には影響しない。「ファイル」メニューに アップロード/ダウンロード。
- **P2P で接続情報を送り合う**(Issue #43、`hashi/p2p.py`)。サーバーを介さず、同じ LAN 内
  などで Hashi 同士が #42 のバンドルを直接転送できる。使い捨て X25519 での ECDH に基づき、
  両者に **6 桁の確認コード(SAS)** を提示して別経路で照合させる(中間者がいると SAS が
  一致せず気づける)。バンドルは共有鍵由来の Fernet で暗号化して送る。秘密情報は #42 と同じく
  パスフレーズ暗号化されているときだけ含まれる。受信は「受信」を開始した瞬間だけポートを開き、
  常時リッスンはしない。ファイル(#42)を手渡しするフォールバックも従来どおり使える。
- **1 接続 = 1 ウィンドウ + 起動時サーバー選択ランチャー**(Issue #14 段階2)。起動画面を
  接続先を選ぶ専用ランチャーにし、接続すると常に独立した SessionWindow が開くようにした
  (タブ方式を廃止)。ポートフォワード / SSH 鍵生成 / サーバー設定変更などの「セッション」
  メニューは各ウィンドウが自前で持つ。プロファイル / 既知ホスト / 認証情報 / 設定は全ウィンドウ
  で共有。ランチャーは「ファイル」メニュー(書き出し/読み込み/P2P/設定)を持ち、セッション
  ウィンドウからは「サーバー一覧を開く」でランチャーへ戻れる。
- **SSH サーバー設定の堅牢化**(Issue #12、`hashi/sshd_admin.py`)。「セッション」メニューから
  接続中のサーバーの **パスワード認証の無効化 / ポート番号の変更** を行える。事故防止のため:
  パスワード無効化は「登録済みの鍵で実際にログインできること」を別接続で検証できた場合のみ実行、
  変更前に sshd 設定をバックアップ、`sshd -t` で構文検証、`sshd_config.d` ドロップイン方式
  (消せば元に戻る)、reload 後に新ポートへの疎通確認、失敗時は自動ロールバック、実行前に
  2 段階確認(確認語入力)。reload は既存接続を切らない(systemd は `systemctl reload`、
  それ以外はマスター sshd のみへ HUP)。IP 固定は別 Issue #45 に分離。
- **接続情報の書き出し / 読み込み**(Issue #42、`hashi/portability.py`)。ファイルメニューから
  プロファイル一式 + known_hosts(ホスト鍵フィンガープリント)を 1 ファイルへエクスポート/
  インポートできる。known_hosts を含めるため移行先でも TOFU の鍵変更警告がそのまま機能する。
  パスワード等の秘密情報は既定で含めず、含める場合はパスフレーズ(scrypt + Fernet)での
  暗号化が必須(平文では絶対に書かない)。インポート時、既存のホスト鍵記録は上書きしない
  (インポートで鍵変更警告を黙らせる事故の防止)。重複プロファイルは上書き / スキップを選択。

### 修正
- **ターミナルを非表示→再表示すると画面内容が数文字に切り詰められるバグ**(Issue #39)。
  切り替え中に Qt が一瞬だけ極小サイズの resizeEvent を配達し、pyte の破壊的リサイズで
  内容が失われていた。リサイズをデバウンスして最終サイズだけを適用するよう修正。
- **パスワード手動送信が「右クリック→送信」の案内どおりに使えなかった**(Issue #40)。
  右クリック=貼り付け(PuTTY 流)のためメニューは Shift+右クリックが必要で、案内と
  実態が食い違っていた。タブ上部に常設の「🔑 パスワード送信」ボタンを追加し(保存済みの
  sudo → ログインの順で送信、未保存なら入力ダイアログ)、トーストの案内文言も実態に修正。

### 追加
- **ProxyJump(踏み台経由の多段接続)に対応**。接続ダイアログの「踏み台 (ProxyJump)」欄
  または `~/.ssh/config` の ProxyJump から指定(OpenSSH 書式 `[user@]host[:port]`、
  カンマ区切りで多段、IPv6 は `[::1]:22`)。踏み台ごとに TOFU のホスト鍵確認と認証
  (config の IdentityFile → エージェント → パスワードの順)を行う。踏み台への
  パスワードプロンプトには「踏み台」を明示し、**接続先の保存済みパスワードを踏み台へ
  自動送信しない/踏み台の秘密は保存しない**。実 sshd 2〜3 台での実機検証済み
  (1 段・2 段チェーン / exec / SFTP 2MB 整合性 / シェル / close 後の全 Transport 解放)。
  `tests/test_proxyjump.py` にユニット + ライブ結合テスト(`HASHI_LIVE_SSH=1`)として恒久化。
- ProxyCommand は引き続き未対応(外部コマンド実行が絡むため)。検出時はエラーで明示し、
  ProxyJump への移行を案内する(黙って直接接続はしない)。
- **ターミナルの代替画面バッファ (smcup/rmcup) に対応** (Issue #6)。`vim` / `htop` / `less`
  などが使う private mode `?1049` / `?1047` / `?47` を解釈し、起動時にメイン画面を退避して
  空の代替画面へ切り替え、終了時に元の画面へ復帰する。`?1049` ではカーソル位置も保存・復元する。
- **ターミナルのマウスレポートに対応**(Issue #6 の残り)。`?1000`(クリック)/
  `?1002`(ドラッグ)/ `?1003`(移動)+ SGR 拡張座標 `?1006` を実装し、vim / htop 内で
  マウスクリック・ドラッグ・ホイールが効くようになった。**Shift 併用でローカル操作
  (選択即コピー / 右クリック貼り付け)に迂回**(xterm 流)。実 sshd + 実 vim で
  クリック位置ジャンプ / ホイールスクロール / 終了後の解除を通し検証済み。
  これで Issue #6 の 3 機能(代替画面 / マウスレポート / ブラケットペースト)が揃った。
